Plano: Migração Python → Rust — fetchless                                                                                                                                                                                                                                │
│                                                                                                                                                                                                                                                                          │
│ Contexto                                                                                                                                                                                                                                                                 │
│                                                                                                                                                                                                                                                                          │
│ O projeto "Token Enhancer" (web-fetch-low-tokens) é um proxy Python que limpa páginas web para reduzir tokens em contextos de agentes AI. A motivação da migração é ter um binário único sem dependências, cross-platform, com CI/CD e instalação com 1 comando. O       │
│ repositório atual (fork do usuário) será reutilizado, substituindo o código Python pelo Rust.                                                                                                                                                                            │
│                                                                                                                                                                                                                                                                          │

│ Ordem de Implementação (por fases)                                                                                                                                                                                                                                       │
│                                                                                                                                                                                                                                                                          │
│ Fase 1 — Scaffold + Fundação                                                                                                                                                                                                                                             │
│                                                                                                                                                                                                                                                                          │
│ 1. Inicializar cargo init --name fetchless                                                                                                                                                                                                                               │
│ 2. Configurar Cargo.toml com todas as dependências                                                                                                                                                                                                                       │
│ 3. config.rs — struct Config com clap (--port, --bind, --db-path, --default-ttl)                                                                                                                                                                                         │
│ 4. error.rs — AppError enum (Validation, Fetch, Database, Parse) + IntoResponse                                                                                                                                                                                          │
│ 5. token.rs — estimate_tokens() + testes inline                                                                                                                                                                                                                          │
│                                                                                                                                                                                                                                                                          │
│ Fase 2 — Lógica Core (testável sem HTTP/DB)                                                                                                                                                                                                                              │
│                                                                                                                                                                                                                                                                          │
│ 6. optimizer.rs — Portar todos os regex patterns, optimize_prompt(), extract_protected(), is_code_or_json(), calculate_confidence(). Usar fancy-regex apenas para TICKER_PATTERN (lookahead). Testes: 7 casos do Python + edge cases.                                    │
│ 7. url_validator.rs — HTTPS enforcement, IP range blocking (127/8, 10/8, 172.16/12, 192.168/16, 169.254/16, 100.64/10, IPv6). DNS async com hickory-resolver. Testes: cada range bloqueado, URLs válidos.                                                                │
│ 8. json_cleaner.rs — Remoção recursiva de chaves junk (meta, ads, tracking, etc). Testes com fixtures.                                                                                                                                                                   │
│ 9. html_cleaner.rs — Scraper: encontrar container (article/main/content div), filtrar noise tags/classes durante walk (scraper é read-only, não muta DOM). Testes com HTML fixtures.                                                                                     │
│                                                                                                                                                                                                                                                                          │
│ Fase 3 — Infraestrutura                                                                                                                                                                                                                                                  │
│                                                                                                                                                                                                                                                                          │
│ 10. db.rs — Todas as operações SQLite consolidadas. 3 tabelas (prompt_log, data_cache, data_log). Arc<Mutex<Connection>> + spawn_blocking para thread safety. Cache get/set com TTL, logging, stats. Testes com tempfile.                                                │
│                                                                                                                                                                                                                                                                          │
│ Fase 4 — Integração                                                                                                                                                                                                                                                      │
│                                                                                                                                                                                                                                                                          │
│ 11. data_proxy.rs — Pipeline: validate URL → check cache → fetch (reqwest) → route to HTML/JSON cleaner → cache set → log. Testes com wiremock.                                                                                                                          │
│ 12. server.rs — Axum router com AppState (db, http_client, config). Endpoints: GET /, POST /refine, POST /fetch, POST /fetch/batch, GET /stats. Manter contrato JSON idêntico ao Python. Testes com axum-test.                                                           │
│ 13. main.rs — Parse config, init tracing, init DB, build reqwest client, mount router, serve. Graceful shutdown com ctrl_c.                                                                                                                                              │
│                                                                                                                                                                                                                                                                          │
│ Fase 5 — MCP Server                                                                                                                                                                                                                                                      │
│                                                                                                                                                                                                                                                                          │
│ 14. mcp.rs — Usar crate rmcp (SDK MCP para Rust). Expor 3 tools: fetch_clean, fetch_clean_batch, refine_prompt. Mesmo formato de resposta do Python.                                                                                                                     │
│                                                                                                                                                                                                                                                                          │
│ Fase 6 — Distribuição                                                                                                                                                                                                                                                    │
│                                                                                                                                                                                                                                                                          │
